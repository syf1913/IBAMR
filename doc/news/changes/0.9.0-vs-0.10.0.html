Incompatibilities

  1. Changed: IBTK_MPI's unused MPI type aliases have been removed. Additionally, the optional communicator arguments have been removed. This change was necessary to prevent a type conversion error that caused compilation issues with versions of MPI that use unsigned int for MPI_Comm.
    (David Wells, 2021/12/15)

Major Changes

  1. New: Implementation of the Immersed Interface Method (IIM) in IBAMR.
    (Amin Kolahdouz and David Wells, 2021/12/17)

Minor Changes

  1. Improved: IBAMR no longer depends on LAPACK.
    (David Wells, 2021/09/03)
  2. Changed: The eel3d example no longer depends on GSL.
    (David Wells, 2021/10/01)
  3. Improved: Classes inheriting from IBStrategy now check, in their constructors, that they are using supported kernels instead of the internal functions checking kernel names every time they are called. This greatly improves performance for applications that call the spread or interpolate functions in loops (e.g., over elements).
    (David Wells, 2021/10/18)
  4. Fixed: The autotools build system now correctly treats tests/attest.conf.in as a configured file. This gets around a problem where this file could get out of date and become invalid when attest was updated but this file was not.
    (David Wells, 2021/10/22)
  5. Added a fully Eulerian version of examples/IBLevelSet/ex0 (a half-buoyant cylinder impacting air-water interface) in examples/IBLevelSet/ex2.
    (Amneet Bhalla, 2021/10/22)
  6. Improved: IBTK::FEProjector now logs solve times for each libMesh system.
    (David Wells, 2021/12/15)
  7. Fixed: IBAMR now uses appropriate comparisons that respect finite precision.
    (Aaron Barrett, 2021/12/23)
