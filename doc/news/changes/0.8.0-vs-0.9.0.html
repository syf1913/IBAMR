Incompatibilities

  1. Removed: IBTK::FEDataManager::buildL2ProjectionSolver() has been removed in favor of using the IBTK::FEProjector class directly.
    (David Wells, 2021/03/30)
  2. Changed: The way IBAMR::IBFEMethod sets up its scratch hierarchy has been completely changed. As a result the exact distribution of work in parallel is now slightly different.
    (David Wells, 2021/05/17)

Major Changes

  1. New: Added an IBLevelSet example of an oscillating cylinder.
    (Kaustubh Khedkar, 2021/03/04)
  2. New: Added IBTK::SecondaryHierarchy, which manages a PatchHierarchy object which occupies the same index space with a different parallel partitioning. Previously this functionality was hard-coded into the IBFEMethod class.
    (David Wells, 2021/05/17)
  3. New: IBMethod and GeneralizedIBMethod now support structural body forces.
    (Boyce Griffith, 2021/05/29)

Minor Changes

  1. Fixed: Zero-displacement constraints added to the finite element matrices now work correctly again.
    (David Wells, 2021/03/01)
  2. Bug fix: Correct the calculations of COM, MOI and associated quantities in the ConstraintIB method in the presence of periodic boundary.
    (Hong Nguyen, 2020/12/01)
  3. Fixed: IBTK::FEMapping (and, consequently, the rest of the IB stack) now works correctly with QNODAL.
    (David Wells, 2021/01/22)
  4. Fixed: IBTK::HierarchyIntegrator now properly traverses child integrators during regridHierarchy().
    (Aaron Barrett, 2021/05/27)
  5. Bug Fix: Filling ghost cells for levelset in demarcateBrinkmanZone function in BrinkmanPenalizationRigidBodyDynamics class.
    (Ramakrishnan Thirumalaisamy, 2021/06/01)
  6. Bug Fix: Computing cell-center locations and cell index using grid_lower_X rather than patch_lower_X in all Brinkman adv diff examples and tests to avoid processor dependent results. New: In IndexUtilities class, new rouine is added to get side-center coordinates using grid_lower_X.
    (Ramakrishnan Thirumalaisamy, 2021/06/21)
  7. New: In IndexUtilities class, new rouine is added to get side-center coordinates using grid_lower_X.
    (Ramakrishnan Thirumalaisamy, 2021/06/21)
  8. Improved: FEMechanicsBase (and therefore IBFEMethod) will evaluate all PK1 functions corresponding to the same registered libMesh systems and quadrature rules inside a single loop, rather than independently looping over each PK1 function independently. This essentially eliminates the overhead of using multiple PK1 stress functions.
    (David Wells, 2021/07/06)
  9. Fixed: FEDataManager now works correctly if all the block ids in a mesh are greater than 256.
    (David Wells, 2021/07/08)
  10. New: GitHub actions are used to build each pull request and push referencing the master branch to verify that (at least on linux) the project builds.
    (David Thompson, 2021/07/12)
  11. New: Configure, build, and test stages are now separate steps in the Github action and submit data to a cdash dashboard.
    (David Thompson, 2021/07/13)
